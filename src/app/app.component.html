Change Detection Strategies Impact on Performance

  • Default Strategy: The default change detection strategy checks for changes in all components whenever any event occurs, which can lead to performance issues in large applications.
  • OnPush Strategy: The OnPush strategy only checks for changes when input properties change or an event occurs within the component, making it more performant by reducing unnecessary checks.

Hierarchical Dependency Injection in Angular

Root Level

Services provided at the root level are singleton instances shared across the entire application.

Parent Level

Services provided at the parent level are specific to the parent and its children, allowing for more fine-grained control and different instances per component tree.

Child Level

Services provided at the child level are specific to the child component, offering the most granular level of dependency injection.

Benefits and Use Cases of Hierarchical DI

  • Scoping Services: Hierarchical DI allows you to scope services to specific parts of your application, reducing memory usage and avoiding unintended shared state.
  • Different Configurations: You can provide different configurations for the same service at different levels of your application.
  • Testing: Hierarchical DI makes it easier to test components in isolation by providing mock services at the component level.
